# Data Model: User CRUD Demo

**Phase**: 1 (Design & Contracts)
**Date**: 2026-06-25

## Entity: User

**Java Implementation**: `@Entity` class with `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` (Lombok). Field validations via Jakarta Validation annotations.

### Table: `users`

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | BIGINT (auto-generated) | PRIMARY KEY, NOT NULL | Auto-generated by JPA |
| username | VARCHAR(30) | UNIQUE, NOT NULL | 3-30 chars, alphanumeric + underscore, immutable after creation |
| first_name | VARCHAR(100) | NOT NULL | 1-100 chars, letters only |
| last_name | VARCHAR(100) | NOT NULL | 1-100 chars, letters only |
| email | VARCHAR(255) | UNIQUE, NOT NULL | Valid email format |
| password | VARCHAR(255) | NOT NULL | BCrypt hash, min 8 chars |
| role | VARCHAR(10) | NOT NULL, CHECK IN ('admin','user') | Default 'user' |
| created_at | TIMESTAMP | NOT NULL | Auto-set on creation |
| updated_at | TIMESTAMP | NOT NULL | Auto-updated on modification |

### Validation Rules (Jakarta Validation)

- `username`: `@NotBlank` + `@Size(min=3, max=30)` + pattern `^[a-zA-Z0-9_]+$`
- `firstName`: `@NotBlank` + `@Size(min=1, max=100)`
- `lastName`: `@NotBlank` + `@Size(min=1, max=100)`
- `email`: `@NotBlank` + `@Email` + `@Size(max=255)`
- `password`: `@NotBlank` + `@Size(min=8, max=100)`
- `role`: `@NotBlank` + must be "admin" or "user"

### Entity Relationships

- **User** has no relationships to other entities (single-domain application).

### State Transitions

- **CREATE**: Admin creates user → all fields mandatory → assigned role
- **READ**: User reads own profile (any role) / Admin reads any profile
- **UPDATE (self)**: User updates firstName, lastName, email, password. Username and role immutable.
- **UPDATE (admin)**: Admin updates any field of any user (including username and role). Only existing admin can grant admin role.
- **DELETE**: Admin deletes any user. Self-deletion terminates admin session.
